WP Manager Pro exposes a secured REST API under the WordPress REST infrastructure. Every feature available in the admin UI — plugins, themes, files, database, users, and more — is accessible programmatically through this API.

## Base URL

```
{site_url}/wp-json/wp-manager-pro/v1
```

Replace `{site_url}` with your WordPress site's URL, for example:

```
https://yoursite.com/wp-json/wp-manager-pro/v1
```

## Authentication

All requests require a valid WordPress nonce generated with the `wp_rest` action. Pass it in the `X-WP-Nonce` request header.

```bash
curl https://yoursite.com/wp-json/wp-manager-pro/v1/dashboard \
  -H "X-WP-Nonce: <nonce_from_wp_localize_script>"
```

See the [Authentication](/api/authentication) page for full details on how to obtain and use the nonce.

## Authorization

Every route requires the `manage_options` capability. Users without this capability — including editors, authors, contributors, and subscribers — receive a `403 Forbidden` response.

## Response format

All responses are JSON. Successful responses include the requested data directly. Error responses always include `code`, `message`, and `data` fields:

```json
{
  "code": "plugin_not_found",
  "message": "Plugin not found.",
  "data": {
    "status": 404
  }
}
```
