---

The security endpoints cover the five-tab Security Suite: admin URL relocation, brute-force login protection, IP blocklisting, WordPress hardening options, core file integrity verification, and TOTP-based two-factor authentication.

## GET /security/overview

Returns a combined snapshot of all security feature states in a single call.

```bash
curl https://yoursite.com/wp-json/wp-manager-pro/v1/security/overview \
  -H "X-WP-Nonce: <nonce>"
```

### Response fields

<ResponseField name="admin_url_enabled" type="boolean">Whether custom login URL protection is active.</ResponseField>
<ResponseField name="admin_url_slug" type="string">The current secret login slug, or empty string if disabled.</ResponseField>
<ResponseField name="custom_url" type="string">The full custom login URL.</ResponseField>
<ResponseField name="login_url" type="string">Standard `wp_login_url()` value.</ResponseField>
<ResponseField name="limiter_enabled" type="boolean">Whether the login attempt limiter is active.</ResponseField>
<ResponseField name="limiter_threshold" type="number">Max failed attempts before lockout.</ResponseField>
<ResponseField name="limiter_window" type="number">Counting window in seconds.</ResponseField>
<ResponseField name="limiter_duration" type="number">Lockout duration in seconds.</ResponseField>
<ResponseField name="lockout_count" type="number">Total number of lockout log entries.</ResponseField>
<ResponseField name="ip_blocklist_count" type="number">Number of IPs currently in the blocklist.</ResponseField>
<ResponseField name="xmlrpc_disabled" type="boolean">Whether XML-RPC is disabled.</ResponseField>
<ResponseField name="hide_wp_version" type="boolean">Whether the WordPress version is hidden from public output.</ResponseField>
<ResponseField name="tfa_enabled" type="boolean">Whether 2FA is enabled for the current user.</ResponseField>
<ResponseField name="wp_version" type="string">Current WordPress version.</ResponseField>
<ResponseField name="wp_locale" type="string">Current site locale.</ResponseField>

## POST /security/admin-url

Enables or updates the custom login URL. Direct GET requests to `wp-login.php` are redirected to the homepage once this is active.

### Request body

<ParamField body="slug" type="string" required>
  Secret login slug (minimum 4 characters). Reserved words such as `wp-admin`, `login`, and `dashboard` are rejected.
</ParamField>

### Response fields

<ResponseField name="success" type="boolean">Always `true` on success.</ResponseField>
<ResponseField name="slug" type="string">The saved slug.</ResponseField>
<ResponseField name="custom_url" type="string">The full custom login URL.</ResponseField>

### Example

```bash
curl -X POST https://yoursite.com/wp-json/wp-manager-pro/v1/security/admin-url \
  -H "X-WP-Nonce: <nonce>" \
  -H "Content-Type: application/json" \
  -d '{"slug": "my-secret-login"}'
```

<Warning>
  Copy the returned `custom_url` before navigating away. If you lose the slug you will be locked out of the default login page.
</Warning>

## DELETE /security/admin-url

Disables admin URL protection and restores access to the default `wp-login.php` path.

```bash
curl -X DELETE https://yoursite.com/wp-json/wp-manager-pro/v1/security/admin-url \
  -H "X-WP-Nonce: <nonce>"
```

### Response

```json
{ "success": true }
```

## POST /security/limiter

Saves login attempt limiter settings.

### Request body

<ParamField body="enabled" type="boolean">
  Enable or disable the login limiter.
</ParamField>

<ParamField body="threshold" type="number" default="5">
  Number of failed attempts within the window before the IP is locked out.
</ParamField>

<ParamField body="window" type="number" default="300">
  Counting window in seconds. Attempts older than this are discarded.
</ParamField>

<ParamField body="duration" type="number" default="900">
  Lockout duration in seconds.
</ParamField>

### Response

```json
{ "success": true }
```

## POST /security/integrity

Fetches official MD5 checksums from `api.wordpress.org` and compares every file in `wp-admin/` and `wp-includes/` against the expected hashes. `wp-content/` is excluded.

```bash
curl -X POST https://yoursite.com/wp-json/wp-manager-pro/v1/security/integrity \
  -H "X-WP-Nonce: <nonce>"
```

### Response fields

<ResponseField name="version" type="string">WordPress version checked against.</ResponseField>
<ResponseField name="locale" type="string">Locale used for the checksum lookup.</ResponseField>
<ResponseField name="checked" type="number">Total number of files checked.</ResponseField>
<ResponseField name="ok" type="number">Files that matched their expected hash.</ResponseField>
<ResponseField name="clean" type="boolean">`true` if no modified or missing files were found.</ResponseField>
<ResponseField name="modified" type="object[]">
  Files whose hash did not match.
  <Expandable title="properties">
    <ResponseField name="modified[].path" type="string">Relative file path.</ResponseField>
    <ResponseField name="modified[].expected" type="string">Expected MD5 hash from wordpress.org.</ResponseField>
    <ResponseField name="modified[].actual" type="string">Actual MD5 hash on disk.</ResponseField>
    <ResponseField name="modified[].size" type="number">File size in bytes.</ResponseField>
    <ResponseField name="modified[].modified" type="number">Last-modified Unix timestamp.</ResponseField>
  </Expandable>
</ResponseField>
<ResponseField name="missing" type="string[]">
  Relative paths of files expected by the checksum list but absent from disk.
</ResponseField>

## POST /security/2fa/setup

Generates a new TOTP secret and QR code URL. The secret is stored as a pending user meta value until confirmed via `POST /security/2fa/verify`.

```bash
curl -X POST https://yoursite.com/wp-json/wp-manager-pro/v1/security/2fa/setup \
  -H "X-WP-Nonce: <nonce>"
```

### Response fields

<ResponseField name="secret" type="string">Base32-encoded TOTP secret. Can be entered manually into any TOTP app.</ResponseField>
<ResponseField name="otp_url" type="string">The full `otpauth://totp/...` URI for QR code generation.</ResponseField>
<ResponseField name="qr_url" type="string">A pre-generated QR code image URL from `api.qrserver.com`.</ResponseField>

---

## POST /security/2fa/verify

Verifies a TOTP code against the pending secret. On success, 2FA is activated and 8 one-time backup codes are generated and returned. The backup codes are shown only once.

### Request body

<ParamField body="code" type="string" required>
  The 6-digit TOTP code from the authenticator app.
</ParamField>

### Response fields

<ResponseField name="success" type="boolean">Always `true` on success.</ResponseField>
<ResponseField name="backup_codes" type="string[]">
  8 one-time backup codes shown exactly once. Store them securely — they are stored as MD5 hashes and cannot be retrieved again.
</ResponseField>

### Example

```bash
curl -X POST https://yoursite.com/wp-json/wp-manager-pro/v1/security/2fa/verify \
  -H "X-WP-Nonce: <nonce>" \
  -H "Content-Type: application/json" \
  -d '{"code": "482916"}'
```
