---
title: "Debug API"
description: "REST endpoints for managing WordPress debug constants and the error log"
---

The debug endpoints read and write WordPress debug constants directly in `wp-config.php` and provide access to the PHP error log. All write operations require `wp-config.php` to be writable by the web server.

## GET /debug

Returns the current state of all debug-related constants and information about the error log file.

```bash
curl https://yoursite.com/wp-json/wp-manager-pro/v1/debug \
  -H "X-WP-Nonce: <nonce>"
```

### Response fields

<ResponseField name="wp_debug" type="boolean">
  Current value of the `WP_DEBUG` constant.
</ResponseField>

<ResponseField name="wp_debug_log" type="boolean">
  Current value of `WP_DEBUG_LOG`.
</ResponseField>

<ResponseField name="wp_debug_display" type="boolean">
  Current value of `WP_DEBUG_DISPLAY`.
</ResponseField>

<ResponseField name="savequeries" type="boolean">
  Current value of `SAVEQUERIES`.
</ResponseField>

<ResponseField name="script_debug" type="boolean">
  Current value of `SCRIPT_DEBUG`.
</ResponseField>

<ResponseField name="log_file" type="string">
  Resolved path to the debug log file (e.g. `/var/www/html/wp-content/debug.log`).
</ResponseField>

<ResponseField name="log_exists" type="boolean">
  Whether the log file currently exists on disk.
</ResponseField>

<ResponseField name="log_size" type="number">
  Size of the log file in bytes. `0` if the file does not exist.
</ResponseField>

<ResponseField name="config_writable" type="boolean">
  Whether `wp-config.php` is writable. If `false`, toggle requests will fail.
</ResponseField>

---

## POST /debug/toggle

Updates one or more debug constants in `wp-config.php`. If a constant does not already exist in the file, it is inserted before the "That's all, stop editing!" line.

<Warning>
  This endpoint writes directly to `wp-config.php`. Setting `WP_DEBUG_DISPLAY` to `true` on a live site will expose errors to all visitors.
</Warning>

### Request body

<ParamField body="wp_debug" type="boolean">
  Set the `WP_DEBUG` constant.
</ParamField>

<ParamField body="wp_debug_log" type="boolean">
  Set the `WP_DEBUG_LOG` constant.
</ParamField>

<ParamField body="wp_debug_display" type="boolean">
  Set the `WP_DEBUG_DISPLAY` constant.
</ParamField>

<ParamField body="savequeries" type="boolean">
  Set the `SAVEQUERIES` constant.
</ParamField>

<ParamField body="script_debug" type="boolean">
  Set the `SCRIPT_DEBUG` constant.
</ParamField>

### Response fields

<ResponseField name="success" type="boolean">
  Always `true` on success.
</ResponseField>

<ResponseField name="message" type="string">
  Confirmation string.
</ResponseField>

<ResponseField name="wp_debug" type="boolean">Saved value for `WP_DEBUG`.</ResponseField>
<ResponseField name="wp_debug_log" type="boolean">Saved value for `WP_DEBUG_LOG`.</ResponseField>
<ResponseField name="wp_debug_display" type="boolean">Saved value for `WP_DEBUG_DISPLAY`.</ResponseField>
<ResponseField name="savequeries" type="boolean">Saved value for `SAVEQUERIES`.</ResponseField>
<ResponseField name="script_debug" type="boolean">Saved value for `SCRIPT_DEBUG`.</ResponseField>

### Example

```bash
curl -X POST https://yoursite.com/wp-json/wp-manager-pro/v1/debug/toggle \
  -H "X-WP-Nonce: <nonce>" \
  -H "Content-Type: application/json" \
  -d '{"wp_debug": true, "wp_debug_log": true, "wp_debug_display": false}'
```

## GET /debug/log

Reads the last N lines of the WordPress error log and optionally filters by PHP log level.

```bash
curl "https://yoursite.com/wp-json/wp-manager-pro/v1/debug/log?lines=100&level=error" \
  -H "X-WP-Nonce: <nonce>"
```

### Query parameters

<ParamField query="lines" type="number" default="200">
  Number of lines to return from the end of the log file.
</ParamField>

<ParamField query="level" type="string">
  Filter log entries by PHP error level. Accepted values: `error`, `warning`, `notice`, `deprecated`. When omitted, all lines are returned.
</ParamField>

### Response fields

<ResponseField name="exists" type="boolean">
  Whether the log file exists.
</ResponseField>

<ResponseField name="content" type="string">
  The raw log content as a single string with newline-separated lines.
</ResponseField>

<ResponseField name="size" type="number">
  Full file size in bytes.
</ResponseField>

<ResponseField name="path" type="string">
  Absolute path to the log file.
</ResponseField>

<ResponseField name="lines" type="number">
  The `lines` value that was requested.
</ResponseField>

<ResponseField name="level" type="string">
  The level filter that was applied, or empty string if none.
</ResponseField>

---

## DELETE /debug/log/clear

Truncates the WordPress error log file to zero bytes. The file itself is preserved; only its content is cleared.

```bash
curl -X DELETE https://yoursite.com/wp-json/wp-manager-pro/v1/debug/log/clear \
  -H "X-WP-Nonce: <nonce>"
```

### Response

```json
{ "success": true, "message": "Error log cleared." }
```

<Info>
  If the log file does not exist, the endpoint returns `success: true` with a message indicating the file was absent. If the file is not writable, a `403` error is returned.
</Info>
