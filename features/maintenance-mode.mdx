Maintenance Mode lets you take your site offline for visitors with a single toggle, while logged-in administrators continue to access the site normally. The maintenance page is a standalone HTML file with fully inline CSS — no external dependencies, no PHP required to display it.

To open Maintenance Mode, click **WP Manager → Maintenance** in the WordPress admin sidebar.

## Enabling and disabling

The status banner at the top of the page shows whether maintenance mode is currently **Active** or **Inactive**.

- Click **Enable** to activate maintenance mode immediately. All visitor traffic is redirected to the maintenance page.
- Click **Disable** to bring the site back online.
- Click **Save Settings** to persist your design changes without changing the active state.

<Warning>
While maintenance mode is active, regular visitors cannot access any content on your site. Only logged-in administrators can view the site normally.
</Warning>

<Note>
The maintenance page file is automatically removed when you disable maintenance mode or deactivate the plugin.
</Note>

## Content tab

Configures the text and status elements shown on the maintenance page.

| Field | Description |
|-------|-------------|
| **Page Title** | Heading displayed on the maintenance page. Defaults to `Site Under Maintenance`. |

Use the colour pickers and hex inputs to fine-tune four individual colour values:

- **Text colour** — the colour of all text on the page

## Access & Extras tab

Configures the countdown timer, role bypass, maintenance scope, and the secret preview URL.

### Countdown timer

Enable **Show Countdown Timer** to display a live ticking countdown on the maintenance page. Set the **Countdown End Time** to a specific date and time. The countdown automatically ticks down to zero in the browser.

If you disable the countdown but still set an end time, it is stored as informational metadata and can be referenced in your message text.

### Bypass roles

Specific WordPress roles can be granted access to the full site during maintenance. Administrators always bypass maintenance mode regardless of this setting.

Check the roles you want to allow through. Users with those roles see the normal site when logged in.

### Maintenance scope

Choose which pages show the maintenance screen:

| Scope | Behaviour |
|-------|----------|
| **Whole Site** | Every page shows maintenance mode. |
| **Home Page Only** | Only the home page is restricted. |
| **Specific Paths** | Restrict access to a list of URL paths you specify. |

When **Specific Paths** is selected, enter one URL path per line. Wildcard `*` is supported, for example `/shop/*`.

### Admin bar toggle

Enable **Show Toggle in Admin Bar** to display a quick maintenance on/off switch in the WordPress admin bar. This lets you toggle maintenance mode without navigating to the settings page.

### Secret bypass URL

Generate a secret key to create a bypass URL that lets anyone with the link preview the full site during maintenance. Visiting the URL sets a 7-day cookie in the browser.

Type a custom word in the key field, or click the refresh icon to auto-generate a random key. Click the copy icon to copy the full bypass URL to your clipboard. Changing the key invalidates any previously shared links. Click **Save Settings** to apply the new key.

## Live preview

The preview pane on the right side of the settings panel updates in real-time as you change any setting. The countdown tiles in the preview tick live in the browser so you can see the exact appearance before saving.

## How it works

When maintenance mode is enabled, WP Manager Pro writes a `wp-content/maintenance.php` file. WordPress checks for this file on every request and serves it to non-admin visitors automatically — this is built-in WordPress behaviour and requires no theme or plugin hooks.

The generated file contains fully inline CSS so it renders correctly in any browser without loading WordPress's stylesheets or scripts.
